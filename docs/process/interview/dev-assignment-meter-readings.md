## История

Василий — энергетик жилого комплекса **«Зелёный квартал»**. Система уже ведёт справочник домов и квартир. Недавно он заказал у фрилансера функцию **«Передача показаний электросчётчика»** — жилец вводит цифры с счётчика, энергетик видит историю и сводку по дому.

Фрилансер сдал работу, получил оплату и пропал. Василий звонит вам:

> «Показания то принимаются, то нет. Обновил цифру за июнь — в таблице старое значение. Декабрь система не принимает. Кнопку „Сохранить" можно нажать дважды — и два раза уходит. Иногда API отвечает 201, а в базе пусто. Разберитесь, пожалуйста, пока жильцы не начали скандалить».

В коде остались комментарии фрилансера (`freelancer`) и вопросы для обсуждения (`TODO [interview]`).

## Ваша задача

За **45 минут** привести передачу показаний в рабочее состояние.

### Жалобы Василия → что проверить

| Жалоба | Где искать |
|--------|------------|
| «Обновил показание — в списке старое» | Handler + Repository + UI |
| «Декабрь не принимает» | Валидация периода |
| «Можно дважды нажать Сохранить» | Frontend, страница показаний |
| «201, но в базе пусто» | Handler, async/await |
| Ложные ошибки «меньше предыдущего» | Handler, бизнес-логика |

### Что сделать

1. **Запустите unit-тесты** handler'а передачи показаний:
   ```powershell
   dotnet test "src\webapi\cnt_gq_webapi\7 Tests\GQ.WebApi.Tests\GQ.WebApi.Tests.csproj" --filter "FullyQualifiedName~SubmitMeterReadingCommandHandlerTests"
   ```
2. **Исправьте** упавшие тесты и связанные баги (ищите комментарии `freelancer`).
3. **Проверьте вручную** (Swagger или UI):
   - POST новое показание → GET история — запись есть;
   - POST то же показание за тот же месяц с большим value → в ответе и в списке **новое** value;
   - POST с `periodMonth: 12` — принимается;
   - на UI кнопка «Сохранить» блокируется на время запроса.
4. **Подготовьте устные ответы** на все `TODO [interview]` в коде (исправлять необязательно, если не успели — обсудим на защите).
